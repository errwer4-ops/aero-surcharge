# Monthly Fuel Surcharge Update Runbook

Last updated: 2026-07-17

Purpose: 매월 한국 출발 국제선 유류할증료 공시가 나올 때마다 같은 순서로 데이터를 검증하고 사이트에 반영하기 위한 작업 백업 문서입니다.

## 1. 업데이트 기준 정하기

공시 반영을 시작하기 전에 아래 기준을 먼저 고정합니다.

- 작업 기준 시각: 예) `2026-07-17 20:55 KST`
- 반영 대상 월: 예) `2026년 8월 유류할증료 공시`
- 다음 전망 대상 월: 예) `2026년 9월 유류할증료 전망`
- 적용 기간: 예) `2026-08-01~2026-08-31`
- 산정 기간: 예) `2026-06-16~2026-07-15`
- 확정 데이터와 전망 데이터를 분리합니다.

중요 원칙:

- 공시 완료 월은 `확정`, `공시 반영`, `사실 데이터`로 표현합니다.
- 다음 월은 공시 전이면 `전망`, `산정 기간 진행 중`, `단계·금액 미확정`으로 표현합니다.
- 미공시 항공사의 금액은 절대 임의 생성하지 않습니다.
- 국내선 유류할증료와 국제선 유류할증료를 섞지 않습니다.

## 2. 공식 공시 자료 수집

항공사별 공식 공지 또는 PDF를 먼저 확인합니다.

확인 항목:

- 항공사명
- 적용 월
- 적용 기간
- 통화 기준: KRW 또는 USD
- 구간별 금액
- 이전 월 대비 변경 여부
- 공식 공지 URL
- PDF 저장 경로
- 발권일 기준인지, 탑승일 기준인지

8월 공시 반영 때 사용한 핵심 확인값:

- 2026년 8월 국제선 유류할증료: 14단계
- 2026년 7월 국제선 유류할증료: 19단계
- 전월 대비: 5단계 인하
- 8월 산정 MOPS 평균: `119.06 USD/bbl`
- 8월 산정 MOPS 평균: `283.48 cents/gal`
- 직전 산정 평균: `142.09 USD/bbl`
- 직전 대비 하락률: 약 `16.2%`

## 4. _shared.js 업데이트

역할:

- 공용 시장 수치
- 공용 i18n 문구
- 항공사별 월별 공시 override
- 계산기/노선조회/항공사 페이지의 기준 데이터

작업 순서:

1. 최신 시장 스냅샷 객체를 추가합니다.
2. `AERO_MARKET_NUMBERS_LATEST`를 새 객체로 교체합니다.
3. 환율이 바뀌면 `window.RATES`의 USD 기준도 갱신합니다.
4. 공용 문구에서 다음 전망 대상 월을 새 월로 수정합니다.
5. 공시 완료 항공사는 해당 월 데이터를 override에 추가합니다.
6. 미공시 항공사는 `공시 예정` 상태로 둡니다.

주의:

- `MOPS 119.06`처럼 공시 산정 평균인 값은 현재 MOPS처럼 쓰지 않습니다.
- 다음 월 산정 평균은 산정 기간 종료 전까지 확정하지 않습니다.

## 5. news.html 업데이트

역할:

- 항공사 공시 소식
- 시장 뉴스
- 공시 완료/미공시 상태 요약
- 전월 대비 변화 요약

작업 순서:

1. 카드뉴스 날짜를 실제 데이터 추가일 기준으로 맞춥니다.
2. 항공사별 공시 카드와 시장 뉴스 카드를 구분합니다.
3. 하나의 카드에 여러 뉴스를 합치지 말고, 뉴스별로 각각 카드화합니다.
4. 공식 공시 섹션을 새 월 기준으로 수정합니다.
5. 전월 대비 섹션을 새 비교 기준으로 수정합니다.
6. 미공시 항공사는 `8월 유류할증료 공시 예정`처럼 짧게 표시합니다.
7. FAQ JSON-LD를 최신 질문으로 갱신합니다.

8월 반영 때 적용한 구분:

- `2026년 8월 공시`: 확정 데이터
- `2026년 9월 전망`: 산정 기간 진행 중
- `호르무즈`, `환율`, `Brent/WTI`: 9월 전망 변수

## 6. forecast.html 업데이트

역할:

- 이미 공시된 월의 결과 요약
- 다음 월 전망
- MOPS, 환율, 국제유가, 호르무즈 변수 설명

작업 순서:

1. 제목과 메타태그를 다음 전망 대상 월로 변경합니다.
2. 페이지 상단에서 확정 월과 전망 월을 분리합니다.
3. 확정 월 단계와 MOPS 산정 평균을 사실 데이터로 표시합니다.
4. 다음 월은 `단계·금액 미확정`으로 표시합니다.
5. FAQ를 다음 전망 대상 월 기준으로 갱신합니다.
6. `article:modified_time`을 작업 기준 시각으로 변경합니다.

예시 문구:

`2026년 8월 국제선 유류할증료는 14단계로 확정된 사실 데이터입니다. 2026년 9월 유류할증료는 산정 기간이 진행 중이므로 단계·금액을 확정하거나 임의 예측 수치로 표시하지 않습니다.`

## 7. 계산기 페이지 업데이트

역할:

- 사용자가 실제 노선별 유류할증료를 비교
- 현재 적용 월과 다음 공시 예정 월 안내

작업 순서:

1. 이전 이전 월 공시는 제거합니다.
2. 현재 적용 중인 공시 월을 기준으로 표시합니다.
3. 다음 월 공시가 일부만 발표되었으면 공시 완료 항공사만 금액 표시합니다.
4. 미공시 항공사는 `공시 예정`으로 표시합니다.
5. 최저 대비/전월 대비 문구가 잘못된 월을 가리키지 않는지 확인합니다.
6. 노선별 그룹 데이터가 새 월 금액을 fallback 없이 가져오는지 확인합니다.

## 8. index.html 업데이트

역할:

- 메인 화면의 최신 공시 상태 요약
- 핵심 CTA
- 항공사 카드 상태

작업 순서:

1. 메인 화면의 월 표시를 새 공시 월로 변경합니다.
2. 전월 대비 문구를 새 비교 기준으로 변경합니다.
3. 모든 항공사 공시가 반영되면 `미공시 항공사 대기` 문구를 제거합니다.
4. 불필요한 공시 현황 섹션은 삭제해도 됩니다.
5. 공식 공지 링크가 새 월 URL로 연결되는지 확인합니다.

## 9. fuel-surcharge-korea.html 업데이트

역할:

- 한국 출발 국제선 유류할증료 전체 비교

작업 순서:

1. 이전 이전 월 데이터는 제거합니다.
2. 현재 적용 월은 유지합니다.
3. 다음 월은 공시 전이면 `공시 예정`으로 표시합니다.
4. 공시 완료 항공사는 실제 금액만 표시합니다.
5. 비교 기준 문구를 `7월 → 8월`처럼 최신 기준으로 수정합니다.

## 10. 항공사별 페이지 업데이트

역할:

- 대한항공, 아시아나, 제주항공 등 개별 항공사 공시 비교

작업 순서:

1. 오래된 비교 월을 제거합니다.
2. 최신 두 달 비교로 전환합니다.
3. 공식 링크를 최신 공지로 변경합니다.
4. 통화 기준을 공시 원문 기준으로 유지합니다.
5. 항공사별 구간 체계를 억지로 통합하지 않습니다.

## 11. 다국어/i18n 검수

언어 변경 시 확인 대상:

- 한국어
- 영어
- 일본어
- 중국어
- 프랑스어
- 독일어

검수 항목:

- nav
- page title
- 요약 박스
- 카드뉴스 제목
- 카드뉴스 설명
- FAQ
- CTA
- 표 헤더
- 공시 예정 문구

주의:

- 한국어/영어만 바뀌고 다른 언어가 깨지는 경우가 많으므로 최종 화면 기준으로 확인합니다.
- 번역이 준비되지 않은 언어는 최소한 영어 fallback이 정상 출력되어야 합니다.
