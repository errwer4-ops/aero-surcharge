# I18N Audit - 2026-07-17

검수 기준: 실제 브라우저 렌더링 기준으로 `en`, `ja`, `zh`, `fr`, `de`를 각각 적용한 뒤 화면에 남는 한글, 언어 선택값, 콘솔 오류를 확인했습니다.

검수 방식:

- Playwright로 각 HTML 페이지를 `http://127.0.0.1`에서 로드
- `localStorage.aero_lang`을 언어별로 설정 후 reload
- `document.documentElement.lang`, `#navLang` 값, 화면 텍스트, 콘솔 오류 확인
- 한글 잔존은 유니코드 코드포인트 `AC00-D7A3` 기준으로 측정

## 요약

전체적으로 한국어/영어 중심 페이지는 많이 정리되어 있지만, 모든 페이지가 언어 설정에 맞게 완전히 출력되는 상태는 아닙니다.

문제 유형:

- 공통 nav가 없는 보조 페이지는 `html lang`이 `ko`로 고정됨
- 동적 렌더링 테이블의 거리 구간/노선 설명이 한국어로 남음
- `forecast.html`의 최신 지표 테이블은 비한국어에서도 한국어 배열을 재사용함
- `news.html`의 일부 카드 링크/태그/공식 공지 문구가 한국어로 남음
- 항공사별 페이지의 CTA, 표 노선명, USD 공시 라벨이 한국어로 남음
- `fuel-surcharge-2026-05.html`은 비한국어에서도 거의 전체가 한국어이고 콘솔 오류가 있음

## 우선 수정 순서

1. `forecast.html`
   - 최신 2026.07.17 override의 `fore.indicator.rows`, `fore.summary.items`, `fore.faq.items`를 언어별로 분리
   - 현재 `en/ja/zh/fr/de`가 한국어 배열을 재사용하는 문제 해결

2. `news.html`
   - 최신 카드뉴스의 `tags`, `links.label`, 공식 공지 문구를 언어별로 분리
   - 항공사명은 고유명사로 유지 가능하지만, `공식 공지`, `유류할증료`, `한국발 국제선`, `인하` 같은 설명 태그는 번역 필요

3. `index.html` + `airlines.html`
   - 메인 카드 CTA, 공시 라벨, 항공사 상태 문구 번역
   - `8월 공식 공지`, `7월 대비 최소`, `공식 공시 반영` 같은 반복 라벨을 공용 번역 키로 이동

4. `fuel-surcharge-calculator.html` + `fuel-surcharge-graph.html`
   - 거리 구간 라벨 번역: `단거리`, `중거리1`, `중거리2`, `장거리`
   - 대표 노선 설명 번역 또는 영어 fallback 적용
   - fr/de에서 `#navLang` 없는 구조 확인

5. 항공사별 페이지
   - 공통 CTA 블록 번역
   - 표의 노선명은 번역 난이도가 높으므로 우선 영어 fallback 또는 로마자/영문 도시명으로 대체
   - USD 공시/공식 라벨 번역

6. 보조 페이지
   - `about`, `methodology`, `privacy`, `terms`는 별도 i18n 구조가 거의 없어 신규 번역 레이어 필요
   - 법적/정책 페이지는 기계 번역보다 영어 fallback을 먼저 만들고, 일본어/중국어/프랑스어/독일어는 단계적으로 확장 권장
