# 언어·통화 전체 검수 결과

- 검수일: 2026-06-06 KST
- 대상: 공개 HTML 17개
- 언어: 한국어, 영어, 일본어, 중국어, 프랑스어, 독일어
- 통화: KRW, USD, JPY, EUR
- 방식: Chromium에서 각 페이지를 열고 선택기 상태, `html[lang]`, 번역 적용, 화면 금액 변화를 자동 비교

## 결론

현재 사이트는 **통화 변환의 핵심 기능은 대체로 정상**이지만, **언어 적용은 페이지별 편차가 크며 전체 페이지 지원 상태는 아니다.**

- 통화 정상: 메인 조회, 계산기, 항공사 인덱스, 그래프, 한국 출발, 5개 항공사 상세
- 통화 오류: `fuel-surcharge-2026-05.html`, `news.html`
- 통화 선택기 혼동: `fuel-surcharge-forecast.html`
- 언어 미지원: About, 데이터 방법론, 개인정보처리방침, 이용약관
- 언어 부분 적용: 항공사 인덱스, 5월 비교, 일부 항공사 상세, 뉴스, 메인 조회
- 프랑스어·독일어 옵션 누락: 5월 비교, 계산기, 그래프, 5개 항공사 상세

## 확인된 공통 원인

1. 일부 페이지의 `navLang` 옵션이 `ko/en/ja/zh`까지만 있어 사이트 전역의 `fr/de` 설정과 불일치한다.
2. 정적 HTML 문구에 `data-i18n`이 없어 언어 변경 시 그대로 남는다.
3. JS로 생성하는 테이블 배지와 링크명이 `공식`, `공시`, `공식 공지`로 하드코딩돼 있다.
4. `fuel-surcharge-2026-05.html`은 `fmtKRW()`를 직접 사용해 `SHARED_STATE.curr`를 무시한다.
5. `news.html`은 `onCurrChange`가 빈 함수이고 금액이 번역 문자열 안에 고정돼 있다.
6. 통화 변환이 필요 없는 전망 페이지에도 통화 선택기가 노출된다.

## 수정 우선순위

### P0

1. `fuel-surcharge-2026-05.html`의 전체 본문을 i18n 키로 연결하고 `fmtAmt()` 기반으로 통화 변환
2. 항공사 상세 공통 배지 `공식/공시`와 원본 통화 배지를 번역 키로 전환
3. About·방법론·개인정보·약관에 공통 언어 상태 적용

### P1

1. `news.html`의 고정 금액을 구조화 데이터로 분리해 선택 통화로 렌더링
2. 모든 동적 페이지의 언어 옵션을 동일하게 맞추거나 지원하지 않는 언어를 전역에서 숨김
3. 항공사 인덱스와 메인 조회의 동적 한국어 문구 번역

### P2

1. 금액 변환이 없는 전망 페이지에서 통화 선택기 제거 또는 “원자료 통화 고정” 표시
2. CI/배포 전 자동 검사에 `scripts/audit_locale_currency.js` 추가
