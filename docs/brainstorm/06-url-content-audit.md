# URL 콘텐츠 감사 회의록 및 실행표

작성 기준: 2026-06-06 KST  
분석 대상: `aero-surcharge.com` 공개 URL, 로컬 배포 파일, GA4 2026-05-08~2026-06-04, Search Console 최신 화면, AdSense 정책 판정

## 1. 현재 상태 요약

- AdSense 소유권 확인은 완료됐으나 `가치가 별로 없는 콘텐츠`로 광고 게재가 준비되지 않은 상태다.
- Search Console 최근 28일 성과는 클릭 185회, 노출 5,320회, CTR 3.5%, 평균 순위 6위다.
- 최근 7일은 클릭 80회, 노출 1,710회, CTR 4.7%로 검색 성장은 분명하다.
- GA4 28일 활성 사용자 771명 중 신규 사용자가 764명이다. 검색 유입은 강하지만 재방문은 약하다.
- 사이트는 조회·계산·전망이라는 독자적 기능을 갖고 있으나, 공개 URL 중 중복·얇은 페이지와 내부 점검 페이지가 함께 노출될 수 있다.
- 검색엔진에 저장된 메인 페이지 내용에는 2026-05-26 기준 문구와 과거 유가 정보가 남아 있어 최신 화면과 검색 스니펫 사이에 시차가 있다.

## 2. 평가 기준

각 URL은 다음 기준으로 평가했다.

| 항목 | 판단 기준 |
|---|---|
| 검색 가치 | 실제 검색 의도가 분명하거나 노출·클릭 성과가 있는가 |
| 사용자 가치 | 사용자가 금액 확인, 비교, 예약 판단 등 구체적 문제를 해결할 수 있는가 |
| 독창성 | 자체 수집 데이터, 계산, 비교, 분석 또는 고유 설명이 있는가 |
| 중복 위험 | 다른 URL과 제목·본문·검색 의도가 겹치는가 |
| 신뢰 | 기준일, 공식 출처, 계산 방법, 검수·수정 정책이 보이는가 |
| 상업성 압력 | 본문 가치에 비해 광고·제휴 유도가 과도하게 보일 가능성이 있는가 |
| 기술 상태 | canonical, redirect, noindex, sitemap, 내부 링크가 올바른가 |

판정:

- **유지·강화**: 색인 유지. 핵심 자산으로 보강한다.
- **유지**: 현재 역할이 명확하며 소폭 정비만 필요하다.
- **통합·리디렉션**: 검색 신호는 보존하되 대표 URL로 합친다.
- **Noindex/차단**: 사용자 검색 결과에 노출될 이유가 없다.
- **삭제/비공개**: 운영·검증 전용 파일로 공개 배포에서 제외한다.

## 3. 팀 회의

### 에단

“검색 노출이 이미 발생하는 URL은 함부로 삭제하면 안 된다. 특히 기존 6월 forecast URL은 자연 검색 노출을 상당히 확보했다. URL은 유지하거나 영구 리디렉션으로 신호를 넘겨야 한다.”

### 아이비

“URL 보존과 동일 문서 두 개를 공개하는 것은 다르다. 두 forecast 파일은 SHA-256 해시까지 완전히 같다. 대표 페이지로 308 또는 301을 일관되게 보내고 응답 경로를 실제 배포에서 검증해야 한다.”

### 노바

“`inspect.html`, `routes.html`, 관리자 테스트 페이지가 공개 배포 안에 있다. `routes.html`은 본문 12자짜리 이동 페이지인데 광고 스크립트까지 포함한다. 이런 파일은 애드센스 심사에서 불필요한 위험이다.”

### 루나

“항공사별 페이지는 사용자 질문이 분명하므로 모두 없애면 안 된다. 다만 현재 다섯 페이지의 정적 본문 유사도가 약 61~83%다. 항공사 이름만 바뀐 템플릿처럼 보이지 않도록 노선군, 통화, 공시 방식, 실제 변경 사례를 각각 고유하게 설명해야 한다.”

### 오리온

“전망 페이지에는 FAQ와 구조화 데이터가 있지만, 코드 안에 날짜별 과거 오버라이드가 대량으로 누적돼 있다. 최종 화면은 최신이어도 검색엔진과 AI가 문서 소스에서 상충하는 문장을 만날 수 있다. 최신 사실과 과거 변경 이력을 분리된 데이터 구조로 제공해야 한다.”

### 아틀라스

“방법론 페이지가 필요하다. MOPS, 국제유가, 환율과 실제 항공사 유류할증료 산정 관계를 어디까지 확인했고 어디부터 자체 전망인지 구분해야 한다. 확정 공시와 전망을 섞지 않는 것이 신뢰의 핵심이다.”

### 베가

“AdSense 승인 전에는 제휴 CTA를 더 늘리면 안 된다. 현재 항공사 페이지는 본문이 약 220~252단어인데 반복된 제휴 관련 코드가 많다. 사용자 결과 뒤 한 번만 노출하고, 숙박 제휴는 여행 비용 문맥이 있는 곳으로 제한해야 한다.”

### 레미

“첫 주에는 새 콘텐츠를 대량 생산하지 않는다. 공개하면 안 되는 URL 정리, 기술 오류 수정, 핵심 네 페이지 보강이 가장 높은 ROI다.”

## 4. 핵심 의견 충돌과 결정

### 충돌 1: 과거 forecast URL 삭제 여부

- 에단: 성과 URL이므로 삭제 반대
- 노바·아이비: 동일 HTML 유지 반대
- 결정: **기존 URL은 영구 리디렉션으로 보존하고 대표 URL만 색인**한다. 리디렉션 체인과 루프를 배포 환경에서 재검증한다.

### 충돌 2: 항공사별 페이지 통합 여부

- 루나: 항공사별 검색 의도가 있으므로 유지
- 아이비: 현재 템플릿 유사도가 높아 저가치 위험
- 결정: **5개 페이지를 유지하되 각각 고유 데이터를 보강**한다. 보강 전에는 제휴 노출을 축소한다.

### 충돌 3: 뉴스·전망 업데이트 빈도

- 에단: 매일 갱신해 신선도 확보
- 아틀라스: 실질 변화 없는 날짜 갱신 반대
- 결정: **판단·수치가 바뀔 때만 본문과 modified time을 갱신**한다. 나머지는 변경 이력으로 누적한다.

### 충돌 4: 승인 전 수익 CTA 유지 범위

- 베가: 고의도 CTA는 유지
- 레미·아이비: 저가치 판정 해소 전 상업성 축소
- 결정: **계산 결과 이후 항공권 CTA 1개만 우선 유지**한다. 뉴스의 숙박 CTA와 반복 CTA는 재심사 전 축소 검토한다.

## 9. 완료 판정 체크리스트

- [ ] 검색 가치 없는 공개 페이지가 모두 비공개·noindex·서버 리디렉션 처리됐다.
- [ ] sitemap, canonical, 내부 링크가 선택한 URL 형식으로 일치한다.
- [ ] robots.txt가 실제 도메인의 sitemap을 가리킨다.
- [ ] 핵심 페이지마다 고유 H1, 기준일, 출처, 방법, 검수일, 수정 정책이 있다.
- [ ] 대표 전망 페이지 소스에 서로 충돌하는 과거 최신 문구가 남아 있지 않다.
- [ ] 항공사별 페이지의 절반 이상이 각 항공사 고유 정보다.
- [ ] 광고·제휴 CTA가 본문보다 먼저 또는 반복적으로 나타나지 않는다.
- [ ] 방법론 페이지가 계산기와 전망 페이지에서 연결된다.
- [ ] 모바일에서 본문과 CTA가 겹치지 않는다.
- [ ] Search Console에서 수정 URL 재색인 후 최신 스니펫을 확인했다.
- [ ] 변경 후 최소 1~2주간 크롤링·사용자 행동을 관찰했다.
- [ ] 위 조건을 충족한 뒤 AdSense 재검토를 요청한다.

## 10. 최종 회의 결정

첫 구현 작업은 콘텐츠를 새로 많이 만드는 것이 아니다.

> 공개하면 안 되는 URL과 중복 문서를 먼저 정리하고, 메인·계산기·전망·그래프가 자체 데이터 플랫폼의 가치를 명확히 증명하도록 만드는 것이 1순위다.

가장 먼저 처리할 개발 묶음:

1. robots Sitemap 수정
2. 점검·관리·얇은 이동 URL 정리
3. forecast 리디렉션 검증 및 중복 제거
4. 핵심 페이지 H1·출처·방법론 연결
5. 방법론 페이지 신설
